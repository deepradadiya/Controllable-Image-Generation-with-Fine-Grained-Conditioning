# Design Document: Demo, Publish & README

## Overview

This feature delivers three interconnected components for the Controllable Image Generation project:

1. **Enhanced Gradio Demo App** вҖ” Upgrades the existing `src/app/gradio_app.py` from a 2-column layout to a 3-panel display (Input Image | Condition Map | Generated Result), adds preset examples, a multi-condition comparison button, and a guidance scale slider with proper range constraints.

2. **publish_to_hub.py Script** вҖ” Replaces the single-model `scripts/deploy_to_hub.py` with a comprehensive publishing pipeline that pushes all three ControlNet adapters to individual repos, creates a combined multi-condition repository, deploys a HuggingFace Space, and generates rich model cards with metrics and visual grids.

3. **Final README.md** вҖ” A polished GitHub README with architecture diagrams, quantitative results tables, visual grids, reproducibility instructions, and links to all published artifacts.

The design leverages existing modules (`ModelManager`, `controls.py`, condition extractors) and extends them rather than replacing them.

- Add a `gr.Examples` component with 6 preset combinations
- Add a "Generate with all 3 conditions" button
- Add a comparison grid output (3Г—2 gallery)

## Correctness Properties

*A property is a characteristic or behavior that should hold true across all valid executions of a system вҖ” essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.*

### Property 1: Guidance Scale Value Passthrough

*For any* valid guidance scale value in the range [1.0, 15.0] with step 0.5, when passed through the Gradio app's generation function, the value SHALL arrive at the inference pipeline unchanged (no clamping, rounding, or transformation).

**Validates: Requirements 3.2**

### Property 2: Multi-Condition Partial Failure Resilience

*For any* single condition type that fails during multi-condition comparison (due to extractor error or generation error), the remaining condition types SHALL still produce their results successfully, and the error SHALL be reported only for the failed type.

**Validates: Requirements 6.6**

### Property 3: Combined Pipeline Directory Organization

*For any* valid set of adapter weight files (1 to 3 condition types present), the combined pipeline organizer SHALL: (a) place each adapter's files in a subdirectory named by its condition type, (b) generate a JSON config that correctly maps each present condition type to its weight file path, and (c) if any adapter files are missing, report an error listing exactly the missing files.

**Validates: Requirements 8.2, 8.3, 8.4**

### Property 4: Model Card Content Completeness

*For any* valid `ModelCardMetadata` instance (with any combination of present/absent FID score, alignment score, and visual grid path), the generated model card SHALL: (a) start with valid YAML front matter containing `license`, `tags`, and `base_model` fields, (b) contain the condition type and base model identifier in the body, (c) include a usage code snippet of 5 lines or fewer containing the repo_id, (d) include each provided metric value, and (e) include a "not yet computed" note for each absent metric.

**Validates: Requirements 10.1, 10.2, 10.4, 10.5**

### Property 5: README Code Block Line Length

*For any* generated README content, all lines within fenced code blocks (``` delimited) SHALL be at most 80 characters in length.

**Validates: Requirements 11.3**

### Property 6: Results Table Structure and Metric Annotation

*For any* valid metrics dictionary containing FID and alignment scores for the three condition types, the generated results table SHALL: (a) contain one row per condition type plus a baseline row, (b) have columns for Model/Condition Type, FID Score, and Alignment Score, and (c) annotate each alignment score with the correct metric name (SSIM for edge, Pearson correlation for depth, normalized keypoint distance for pose).

**Validates: Requirements 12.1, 12.3**

### Property 7: README Header Structure

*For any* generated README, the first line SHALL be a Markdown level-1 heading (`# `) containing the project title, and the first non-empty line after the title SHALL be a plain-text description of at most 200 characters.

**Validates: Requirements 16.1, 16.3**

### Property 8: Model Repository URL Formation

*For any* valid HuggingFace username and condition type, the generated model repository URLs SHALL follow the pattern `https://huggingface.co/{username}/controlnet-sd15-{condition_type}` and all three condition type URLs SHALL be present in the links section.

**Validates: Requirements 15.1**

## Error Handling

### Gradio App Errors

| Error Scenario | Handling Strategy |
|---|---|
| Image upload fails | Display error in Status textbox, keep panels empty |
| Condition extraction fails | Show error in Status, leave Condition Map panel empty, allow retry |
| Generation fails | Show error in Status, leave Generated Result empty, keep condition map visible |
| Model loading fails (OOM) | Display user-friendly memory error via `ModelManager._format_user_error()` |
| Preset image file missing | Show error indicating which preset is unavailable, keep inputs unchanged |
| Multi-condition partial failure | Continue processing other types, show error in failed type's grid cell |
| Network timeout during model download | Display connection error with retry suggestion |

### Publishing Script Errors

| Error Scenario | Handling Strategy |
|---|---|
| No valid HF token | Exit with code 1, print auth failure message |
| Adapter weight files missing | Abort upload, list all missing files in error message |
| Network failure during upload | Exit with code 1, print network error with retry suggestion |
| Repository creation fails (quota) | Exit with code 1, print quota/permission error |
| Upload verification fails | Log warning, continue with next adapter (non-fatal) |
| Space deployment fails | Exit with code 1, print specific failure reason |
| Metrics file not found | Generate model card without metrics, include "not computed" note |
| Visual grid image not found | Generate model card without image, include placeholder text |

### README Generation Errors

| Error Scenario | Handling Strategy |
|---|---|
| Metrics unavailable | Generate table with "N/A" placeholders |
| Visual grid images missing | Include placeholder text instead of image references |
| Invalid metrics values | Skip invalid values, note them as "pending" |
